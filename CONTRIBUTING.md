# Contributing to AdShield CoreKit

Thank you for considering a contribution! We welcome improvements to the benchmark engine, UI/UX, documentation, and deployment tooling.

## Getting started
1. Fork the repo and create a new branch.
2. Install dependencies with `npm install`.
3. Run `npm run dev` for local development; `npm run build` to verify production output.
4. Please include tests or reproduction steps for bug fixes when possible.

## Coding standards
- React + TypeScript with strict mode.
- Follow ESLint/Prettier rules included in the project.
- Avoid introducing heavy dependencies unless they provide clear value.
- Keep UI responsive and accessible.

## Pull requests
- Clearly describe the change and reasoning.
- If the UI changes, consider adding before/after screenshots.
- Document limitations or trade-offs in the PR description.

## Reporting issues
- Provide the URL you attempted to benchmark and any console errors.
- Mention your browser and OS.

Thanks for helping us build a better, privacy-friendly benchmarking toolkit!
